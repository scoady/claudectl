# v2.0.1

## Summary

This release focuses on architecture and maintainability without backing out the current workspace shell experience.

## Included

- split workspace-specific app update flow into dedicated files:
  - `app_data.go`
  - `app_tools_update.go`
  - `app_overlays.go`
  - `app_workspace.go`
  - `app_workspace_keys.go`
  - `app_workspace_update.go`
- split workspace shell concerns into dedicated files:
  - `workspace_shell_explorer.go`
  - `workspace_shell_git.go`
  - `workspace_shell_layout.go`
  - `workspace_shell_presenters.go`
  - `workspace_shell_transcript.go`
  - `workspace_shell_types.go`
- introduced a canonical transcript row model and focused transcript tests
- consolidated workspace git state into a single git context model
- documented modularization, reuse, object-oriented design, and minimum testing expectations in repo standards
